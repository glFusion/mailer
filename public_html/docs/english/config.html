Mailer Plugin

The Mailer Plugin allows site visitors to subscribe to announcements and other communications. This serves as a one-way mailing list. Unlike the built-in "Mail Users" function, non-members can also subscribe to mailings.

Mailer Plugin Configuration Options

Main Settings
Option Description
Limit for subscribers to confirm Enter the number of days to wait for a subscriber to confirm their subscription. Subscriptions left unconfirmed after this period will be automatically deleted. Enter zero for no limit.
Default days to keep messages Enter the number of days that mailers will be retained by default. Each mailer may have its own retention value. Mailers older than this number of days will be automatically deleted. Enter zero to disable purging.
Mail Queue Settings
Option Description
Max. items mailed at once Enter the maximum number of messages to be sent in one batch. Zero indicates that all queued messages will be sent. This is intended to help compliance with outbound mail restrictions.
Delay between queue processing Enter the number of seconds between mailings. The emails are sent during glFusion's cron process, so this value should be greater than that interval to be meaningful. Zero indicates that up to the maximum number of messages will be sent with each scheduled task run.
