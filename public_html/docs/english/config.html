glFusion Mailer Plugin Configuration

The Mailer Plugin allows site visitors to subscribe to announcements and other communications. This serves as a one-way mailing list. Unlike the built-in "Mail Users" function, non-members can also subscribe to mailings.

Main Options

List Provider
Select the mailing list provider that you use.
Subscription Confirmation Time Limit
Enter the number of days to wait for a subscriber to confirm their subscription. Subscriptions left unconfirmed after this period will be automatically deleted. Enter zero for no limit.
Default days to keep messages
Enter the number of days that mailers will be retained by default. Each mailer may have its own retention value. Mailers older than this number of days will be automatically deleted. Enter zero to disable purging.
Max. items mailed at once
Enter the maximum number of messages to be sent in one batch. Zero indicates that all queued messages will be sent. This is intended to help compliance with outbound mail restrictions.
Delay between queue processing
Enter the number of seconds between mailings. The emails are sent during glFusion's cron process, so this value should be greater than that interval to be meaningful. Zero indicates that up to the maximum number of messages will be sent with each scheduled task run.
Sender Address
Select the address to show as "From"
Days to keep messages
Enter a number of days to keep messages available online. Enter 0 to disable purging.
Unsubscribe deleted users
Select whether to automatically unsubscribe users from mailing lists when they are deleted from the site.
Require double opt-in for members
Select whether to require double opt-in for site members when they subscribe via their user preferences.
At-registration subscriptions
Select how, or if, new users can opt into the mailing list.
Log Level
Select the log level to use. Enable debug logging to log most events if you are having issues.
Show signup block to subscribers?
The signup block can be hidden from logged-in users who are already subscribed. Select "No" to display the block anyway. Anonymous users always see the block if it is enabled.
Mailchimp API Key
Enter your API key from Mailchimp.
Mailchimp Default List ID
Enter the list ID from Mailchimp for your default email list (audience)
Mailchimp Firstname merge field ID
Enter the ID of the merge field used for "First Name"
Mailchimp Lastname merge field ID
Enter the ID of the merge field used for "Last Name"
Sendinblue API Key
Enter your API key from Sendinblue.
Sendinblue Default List ID
Enter the list ID from Sendinblue for your default email list.
Sendinblue Double Opt-In Template ID
Enter the ID number for the double opt-in template.
